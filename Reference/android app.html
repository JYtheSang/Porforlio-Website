UI Flow

Activity Navigation

Six Activities. All navigation routes back through MainMenu. The Compare Jobs button is conditionally enabled based on a data count check at runtime.

1
MainMenu.java

Launches each Activity via Intent. Queries job count to enable/disable Compare button.

2
current_job_activity

Pre-populates form from DB if a current job exists. Writes to jobs table on save.

3
job_offer_activity

Inserts new row into jobs table. Optionally launches compare result directly.

4
comparison_settings

Reads/writes the single row in comparison_settings table via ComparisonSettings singleton.

5
compare_jobs

Calls JobComparison.getRankedJobs() to sort and display. Passes two selected IDs to result.

6
compare_result

Reads ComparisonData from singleton. Renders a 10-field comparison table.

Architecture

App Structure

Three layers: Activities own UI and navigation logic, domain Classes hold business logic and are accessed as singletons, and Database helpers wrap SQLite via SQLiteOpenHelper.

UI Layer
MainMenu.java current_job_activity.java job_offer_activity.java comparison_settings.java compare_jobs.java compare_result.java
Domain Layer
Job.java JobComparison.java ComparisonSettings.java ComparisonData (inner)
Data Layer
JobDatabase.java JobDatabaseHelper.java ComparisonSettingsDatabaseHelper.java SQLite (on-device)

Domain Classes

Job.java

Plain Java model. Exposes computed getters alongside raw fields: calculateAdjustedYearlySalary() and calculateAdjustedYearlyBonus() apply the CoL index; calculateHealthInsurance() adds 2% of adjusted salary; calculateEmployeeDiscount() clamps the value to min(input, 0.18 × salary).

JobComparison.java

Application-scoped singleton (getInstance(Context)). Accepts a job list and a weight list via setters. rankJobs() builds an index list then sorts it by score using Comparator.comparingDouble(...).reversed(). compareJobs() populates an inner ComparisonData POJO shared to the result Activity.

ComparisonSettings.java

Application-scoped singleton. Wraps ComparisonSettingsDatabaseHelper. On first instantiation, checks row count — inserts default row (all weights = 1) if empty. Exposes getSettings() returning an ArrayList<Integer> from a Cursor.

Scoring Formula

Weighted Scoring Implementation

calculateScore(Job) in JobComparison.java computes a double score per job. Each term divides its weight by 9 (max weight) so the scale is invariant to the absolute weight values.

// Weighted Job Score
score =
  (salaryWeight / 9) × adjustedYearlySalary
+ (bonusWeight / 9) × adjustedYearlyBonus
+ (tuitionWeight / 9) × tuitionReimbursement
+ (healthWeight / 9) × (healthInsurance + adjustedSalary × 0.02)
+ (discountWeight / 9) × employeeDiscount
+ (adoptionWeight / 9) × (childAdoptionAssistance / 5)

// Cost-of-living adjustment applied to salary and bonus
adjustedSalary = yearlySalary × (100 / costOfLivingIndex)
adjustedBonus = yearlyBonus × (100 / costOfLivingIndex)

rankJobs() builds a parallel index list, sorts it with Comparator.comparingDouble(scores::get).reversed(), then writes sorted Job objects into rankedJobs. The isCurrentJob flag on the model drives the label in the list adapter.

Features

Implemented Functionality

Five functional areas, each mapping to one or more Activities and domain methods.

1
Current Job — Enter & Edit
current_job_activity loads existing data via JobDatabaseHelper.getCurrentJob() on launch and pre-fills all 10 fields. On save, client-side validation runs before calling insert() or update() depending on whether a record already exists.
2
Job Offers — Insert & Shortcut Compare
job_offer_activity always inserts a new row (isCurrentJob = false). After a successful save, the activity checks for an existing current job; if found, it enables an inline "Compare with Current Job" button that launches compare_result directly.
3
Comparison Settings — Persistent Weights
comparison_settings reads the singleton's weight array into six input fields. On save, each field is validated as an integer in [1, 9] before ComparisonSettings.setSettings(...) writes the updated row to SQLite.
4
Ranked Job List
compare_jobs fetches all jobs from JobDatabaseHelper, passes them and the current weights into JobComparison.setJobList() and setSettings(), then calls getRankedJobs(). The result drives a ListView adapter that marks the current job entry.
5
Side-by-Side Comparison Table
compare_result calls JobComparison.compareJobs(jobA, jobB), then reads the populated ComparisonData POJO to fill a 10-row table: Title, Company, Location, Adj. Salary, Adj. Bonus, Tuition, Health Insurance, Employee Discount, Child Adoption Assistance, Job Score.

Implementation Notes

Validation & Edge Cases

Input constraints are enforced at the Activity layer before any write to SQLite.

Employee discount is silently clamped to min(input, 0.18 × yearlySalary) in the Job constructor
Health insurance stored as the raw premium; the 2% salary surcharge is computed at score time via calculateHealthInsurance()
Weights validated as integers in [1, 9] before the settings UPDATE is issued
Compare Jobs button state is recalculated in MainMenu.onResume() — queries JobDatabaseHelper for offer count and current-job flag
Child adoption assistance term divided by 5 in the score formula to normalize its scale relative to other monetary fields
ComparisonSettings and JobComparison use synchronized getInstance(Context) — safe for single-threaded Activity lifecycle but protects against accidental double-init
